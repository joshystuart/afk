**1. Create `server/src/gateways/session-gateway-terminal.service.ts`:**

Follow the exact pattern of `SessionGatewayChatService`. The service must:

**Constructor dependencies (via DI):**
- `SessionGatewaySubscriptionsService` — for `recordSessionActivity`
- `SessionSubscriptionService` — for `getSessionsForClient` (cleanup on disconnect)
- `DockerEngineService` — for `execInteractive`
- `SessionRepository` via `@Inject(SESSION_REPOSITORY)` — to look up container ID and repo URL
- `SessionIdDtoFactory` — to convert string session IDs to typed IDs
- `Logger` — `new Logger(SessionGatewayTerminalService.name)`

**Internal state:**
- `private activePtys = new Map<string, Map<string, { stream: NodeJS.ReadWriteStream; execId: string; resize: (...) => Promise<void>; destroy: () => void }>>()` — keyed by `sessionId` → `clientId` → PTY session. One active PTY per session per client (T-1-03 DoS mitigation).

**Methods:**

`async handleTerminalStart(server: Server, client: Socket, data: { sessionId: string; cols: number; rows: number })`:
- **Security (T-1-01):** Check that client.id is subscribed to the session by calling `this.sessionSubscriptionService.getSubscribersForSession(data.sessionId)` and verifying `client.id` is in the array. If not, return `{ event: SOCKET_EVENTS.terminalError, data: { sessionId, error: 'Not subscribed to session' } }`.
- **Security (T-1-04):** Validate `cols` is integer between 1 and 500, `rows` is integer between 1 and 200. If invalid, return error event.
- **DoS mitigation (T-1-03):** If client already has an active PTY for this session, destroy the old one before creating a new one.
- Look up session from repository to get `containerId` and `repoUrl`. If no container, return error.
- Determine working dir: `getExecWorkingDir(session.repoUrl)` from `docker.constants.ts`.
- Call `this.dockerEngine.execInteractive(session.containerId, cols, rows, workingDir)`.
- Store the PTY session in `activePtys` map under `sessionId` → `client.id`.
- Set up data listener on the stream: `ptySession.stream.on('data', (chunk: Buffer) => { client.emit(SOCKET_EVENTS.terminalData, { sessionId, data: chunk.toString('base64') }) })`. Use base64 encoding for binary safety.
- Set up end/close listener: clean up from `activePtys`, emit `terminal.close` to client.
- Record session activity.
- Return `{ event: SOCKET_EVENTS.terminalStarted, data: { sessionId } }`.

`handleTerminalInput(client: Socket, data: { sessionId: string; data: string })`:
- **Security (T-1-02):** Verify client is subscribed (same check as above). Reject if not.
- Look up PTY session from `activePtys[sessionId][client.id]`. If not found, return error.
- Decode base64 input and write to stream: `ptySession.stream.write(Buffer.from(data.data, 'base64'))`.

`async handleTerminalResize(client: Socket, data: { sessionId: string; cols: number; rows: number })`:
- **Security (T-1-04):** Validate cols 1-500, rows 1-200.
- Look up PTY session. Call `ptySession.resize(cols, rows)`.

`handleTerminalClose(client: Socket, data: { sessionId: string })`:
- Look up and destroy PTY session. Remove from `activePtys`.

`async handleDisconnect(clientId: string)`:
- Iterate all sessions in `activePtys`. For each session that has an entry for `clientId`, call `destroy()` and remove the entry. This prevents zombie exec processes (Pitfall 5 from RESEARCH.md).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Socket.IO gateway | Untrusted terminal input (keystrokes, resize dimensions) crosses from browser to server |
| Gateway → Docker exec | Server creates a PTY inside the container — must verify the requesting client owns the session |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Tampering / Elevation | SessionGatewayTerminalService.handleTerminalStart | mitigate | Verify client.id is in `getSubscribersForSession(sessionId)` before creating PTY. Reject with error event if not subscribed. |
| T-1-02 | Tampering | SessionGatewayTerminalService.handleTerminalInput | mitigate | Verify client.id has an active PTY for the session before writing to stream. Reject input from non-subscribed sockets. |
| T-1-03 | Denial of Service | SessionGatewayTerminalService activePtys | mitigate | Limit one active PTY per session per client. Destroy existing PTY before creating replacement. Clean up all PTYs on disconnect. |
| T-1-04 | Denial of Service | SessionGatewayTerminalService.handleTerminalResize | mitigate | Validate cols (1-500) and rows (1-200) as positive integers before calling exec.resize(). Reject oversized values. |
</threat_model>
