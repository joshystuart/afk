**3. Create `web/src/hooks/useTerminal.ts` (per D-03, D-16):**

Follow the `useChat` hook pattern for socket event management. The hook:

- Accepts `sessionId: string`
- Gets `{ socket, connected }` from `useWebSocket()`
- Tracks internal state: `status` enum of `'idle' | 'connecting' | 'connected' | 'disconnected' | 'error'`, `reconnectAttempt: number`
- Provides these functions via `useCallback`:
  - `startTerminal(cols: number, rows: number)`: emits `'terminal.start'` with `{ sessionId, cols, rows }`. Sets status to `'connecting'`.
  - `sendInput(data: string)`: emits `'terminal.input'` with `{ sessionId, data }` where `data` is base64-encoded
  - `resize(cols: number, rows: number)`: emits `'terminal.resize'` with `{ sessionId, cols, rows }`
  - `closeTerminal()`: emits `'terminal.close'` with `{ sessionId }`. Sets status to `'idle'`.
- Accepts an `onData` callback ref (via `useRef`) that receives base64-encoded terminal output data
- Sets up socket listeners in a `useEffect` (following useChat pattern):
  - `'terminal.started'`: set status to `'connected'`, reset reconnectAttempt to 0
  - `'terminal.data'`: filter by sessionId, call `onDataRef.current(payload.data)` with the base64 data
  - `'terminal.error'`: set status to `'error'`, log error
  - `'terminal.close'`: set status to `'disconnected'`
- Cleanup: remove all listeners on unmount
- **Auto-reconnect (D-16):** When socket reconnects (track `connected` changes from false→true) and status was `'connected'` or `'disconnected'`, re-emit `'terminal.start'` with last known cols/rows (store in ref). Max 3 reconnect attempts, then set status to `'error'`.
- Returns `{ startTerminal, sendInput, resize, closeTerminal, status, setOnData }` where `setOnData` is a function to update the onData ref

**1. Create `web/src/components/session/TerminalView.tsx` (per D-01, D-05, D-17):**

A React component that wraps xterm.js. Props: `{ sessionId: string; visible: boolean }`.

Implementation:
- Import `Terminal` from `@xterm/xterm`, `FitAddon` from `@xterm/addon-fit`, `WebLinksAddon` from `@xterm/addon-web-links`, and `@xterm/xterm/css/xterm.css` (Pitfall 1 from RESEARCH.md).
- Use `useTerminal(sessionId)` hook for socket lifecycle.
- Use `useSessionHealth(sessionId, true)` to check if session/terminal is ready.
- Refs: `containerRef` for the div, `terminalRef` for Terminal instance, `fitAddonRef` for FitAddon, `initializedRef` boolean to prevent double-open (Pitfall 4).

**Terminal initialization (in useEffect, runs once):**
- Guard: if `!containerRef.current || initializedRef.current` return.
- Create `Terminal` with these exact options (per UI-SPEC.md):
  ```
  cursorBlink: true
  cursorStyle: 'bar'
  fontSize: 14
  fontFamily: "'JetBrains Mono', 'Menlo', monospace"
  theme: {
    background: '#09090b',
    foreground: '#fafafa',
    cursor: '#10b981',
    selectionBackground: 'rgba(16, 185, 129, 0.3)',
  }
  scrollback: 5000
  allowProposedApi: true
  ```
- Load FitAddon and WebLinksAddon.
- Call `terminal.open(containerRef.current)`.
- Call `fitAddon.fit()`.
- Wire `terminal.onData((data) => sendInput(data))` — when user types in xterm, send to server. Note: xterm's `onData` provides raw string data, encode to base64 before sending.
- Set `onData` callback on useTerminal to decode base64 and write to terminal: `terminal.write(Buffer.from(data, 'base64'))`. Use `atob` + `Uint8Array` for browser compatibility instead of Node's Buffer.
- Set `initializedRef.current = true`.
- Cleanup on unmount: `terminal.dispose()`, reset `initializedRef`.

**Fit on visibility change (per Pitfall 6):**
- `useEffect` watching `visible`: when `visible` becomes true, call `requestAnimationFrame(() => fitAddonRef.current?.fit())`. Then emit resize with the new dimensions from `fitAddon.proposeDimensions()`.

**ResizeObserver for dynamic resize (per D-05, Pitfall 2):**
- Set up `ResizeObserver` on `containerRef.current`. On resize, debounce `fit()` by 50ms. After fit, emit `resize(cols, rows)` to server.

**Terminal start trigger:**
- When `visible` is true AND terminal is initialized AND `useSessionHealth` reports `terminalReady` AND `useTerminal.status === 'idle'`, call `startTerminal(cols, rows)` with dimensions from `fitAddon.proposeDimensions()`.

**State display (per UI-SPEC.md Terminal Panel States):**
- `status === 'connecting'`: Show centered MUI `CircularProgress` (accent color) + "Connecting to terminal..." caption (`#a1a1aa`, 12px)
- `terminalReady === false`: Show centered terminal icon (`#71717a`, 48px) + "Session not running" heading (13px, `#a1a1aa`) + "Start the session to access the terminal" body (12px, `#71717a`)
- `status === 'disconnected'`: Inline banner at top: "Terminal disconnected — reconnecting..." with accent spinner
- `status === 'error'`: Inline banner: "Terminal connection failed — check that the session is running and try again" with `#ef4444` icon
- `status === 'connected'`: xterm.js terminal fills the container

**Render:**
- Outer `Box` with `width: '100%', height: '100%', position: 'relative', bgcolor: '#09090b'`
- Status overlays rendered conditionally above the terminal div
- Terminal div: `ref={containerRef}` with `width: '100%', height: '100%'`
