    **4. Create `workspace-file-listing.service.ts`:**
    Injectable service with constructor injection of `DockerContainerExecService`.

    **`resolveSafePath(workspaceBase: string, userPath: string): string`** — Use `path.resolve(workspaceBase, userPath)` then verify result starts with `workspaceBase`. Throw `BadRequestException('Path traversal detected')` if it escapes. Import `BadRequestException` from `@nestjs/common`.

    **`listDirectory(containerId: string, dirPath: string, workingDir: string): Promise<FileEntryDto[]>`** —
    First call `resolveSafePath(workingDir, dirPath)` to validate path.
    Try `git ls-files` first: run `['git', 'ls-files', '--cached', '--others', '--exclude-standard', '--directory', resolvedPath + '/']` with `-z` flag for null-byte separator. Parse only immediate children of the resolved path (depth 1). If git exits non-zero, fall back to find.

    Fallback: run `['find', resolvedPath, '-maxdepth', '1', '-mindepth', '1']` then filter results using the `ignore` npm package. Load `.gitignore` from workspace root via `execInContainer` (`['cat', path.join(workingDir, '.gitignore')]`), create an `ignore()` instance and filter the find results.

    For each entry, determine type by running `['test', '-d', entryPath]` (exit 0 = directory). Return `FileEntryDto[]` sorted: directories first, then alphabetical within each group.

    **NOTE on portable listing:** For directory listing specifically (not the flat index), an alternative simpler approach: run `['sh', '-c', 'ls -1Ap "' + safePath + '"']` which uses `-p` to append `/` to directories. Or run `['stat', '--printf=%F %n\n', resolvedPath + '/*']`. Choose whichever is most reliable inside the AFK Docker images. The key requirements are: (a) immediate children only, (b) distinguish files from directories, (c) respect .gitignore. A pragmatic approach: use `git ls-tree HEAD --name-only <path>` for git repos (inherently .gitignore-aware, returns type info) with fallback to `find -maxdepth 1` + ignore filtering.

    **`getFileContent(containerId: string, filePath: string, workingDir: string): Promise<FileContentResponseDto>`** —
    Validate path with `resolveSafePath`. Check binary by extension against a set: `['.png', '.jpg', '.jpeg', '.gif', '.ico', '.woff', '.woff2', '.ttf', '.eot', '.pdf', '.zip', '.tar', '.gz', '.mp3', '.mp4', '.mov', '.avi', '.exe', '.dll', '.so', '.dylib', '.o', '.a']`. If binary, return `{ binary: true, content: '', ... }`.
    Get file size: `['stat', '-c', '%s', resolvedPath]`. If size > 524288 (512KB), read only first 512KB via `['head', '-c', '524288', resolvedPath]` and set `truncated: true`. Otherwise read via `['cat', resolvedPath]`. Derive language from extension using a mapping: `.ts`→`typescript`, `.tsx`→`tsx`, `.js`→`javascript`, `.json`→`json`, `.css`→`css`, `.html`→`html`, `.py`→`python`, `.sh`→`bash`, `.yml`/`.yaml`→`yaml`, `.md`→`markdown`, `.rs`→`rust`, `.go`→`go`, `.java`→`java`, `.sql`→`sql`, default→`plaintext`.

    **Private cache:** `Map<string, { files: string[]; timestamp: number }>` keyed by containerId.

    **`getFileIndex(containerId: string, workingDir: string): Promise<string[]>`** — Check cache; if valid (within TTL), return cached. Otherwise build: run `['git', 'ls-files', '--cached', '--others', '--exclude-standard']` in workingDir. Split stdout by newline, filter empty. If git fails (exit != 0), fall back to `['find', '.', '-type', 'f', '-not', '-path', './.git/*']` with `ignore` package filtering. Slice to MAX_INDEX_SIZE. Cache and return.

    **`clearAllIndexes(): void`** — Clear entire cache map (for cleanup on session delete).

    Add `private readonly logger = new Logger(WorkspaceFileIndexService.name)`.
