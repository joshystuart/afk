## Trust Boundaries

| Boundary                 | Description                                                             |
| ------------------------ | ----------------------------------------------------------------------- |
| User input → file search | @ character followed by filter text triggers client-side Fuse.js search |
| Settings form → server   | IDE command value submitted to Settings update API                      |

## STRIDE Threat Register

| Threat ID | Category               | Component                 | Disposition | Mitigation Plan                                                                                                                                                                                                                                                     |
| --------- | ---------------------- | ------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| T-03-08   | Spoofing               | FileAutocomplete          | accept      | Autocomplete displays server-provided file paths; no user-controlled content is rendered as HTML. Paths are plain text in the prompt input.                                                                                                                         |
| T-03-09   | Tampering              | ideCommand Settings field | accept      | ideCommand is a simple string stored in Settings entity with varchar(100) length limit. It's used client-side only to construct protocol URLs — the server does not execute it. Validation is not security-critical since the user configures it for their own IDE. |
| T-03-10   | Information Disclosure | File index exposure       | accept      | The file index contains relative paths within the workspace — no secrets. The index is only available to authenticated users with access to the session. Same auth boundary as the session itself.                                                                  |

</threat_model>
